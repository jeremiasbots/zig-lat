  <h1 class="text-white py-8">Asignacion</h1>

  La asignación de valores tiene la siguiente sintaxis: <span class="bg-gray-800 rounded-md px-2 text-purple-400">(const|var) identificador[: type] = valor</span>.

  <ul class="ml-10 list-disc">
    <li><span class="bg-gray-800 rounded-md px-2 text-purple-400">"const"</span> indica que el identificador es una constante que almacena un valor inmutable.</li>
    <li><span class="bg-gray-800 rounded-md px-2 text-purple-400">"var"</span> indica que el identificador es una variable que almacena un valor mutable.</li>
    <li><span class="bg-gray-800 rounded-md px-2 text-purple-400">": type"</span> es una anotación de tipo para el identificador y puede ser omitida si el <span class="bg-gray-800 rounded-md px-2 text-purple-400">type</span> de dato del <span class="bg-gray-800 rounded-md px-2 text-purple-400">valor</span> puede ser inferido.</li>
  </ul>

  <Prism lang="zig" code={`const constant: i32 = 5; // constante con signo de 32 bits
var variable: u32 = 5000; // variable sin signo de 32 bits

// @as realiza una coerción de tipo explícita
const inferred_constant = @as(i32, 5);
var inferred_variable = @as(u32, 5000);`}/>

Las constantes y variables deben tener un valor. Si no se puede proporcionar un valor conocido, se puede utilizar el valor <span class="bg-gray-800 rounded-md px-2 text-purple-400">undefined</span>, el cual se convierte a cualquier tipo, siempre y cuando se proporcione una anotación de tipo.
<Prism lang="zig" code={`const a: i32 = undefined;
var b: u32 = undefined;`}/>

Cuando sea posible, se prefieren los valores constantes <span class="bg-gray-800 rounded-md px-2 text-purple-400">(const)</span> sobre los valores variables <span class="bg-gray-800 rounded-md px-2 text-purple-400">(var)</span>.
